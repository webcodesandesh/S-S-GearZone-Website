  ✅  OrderController    — place order, list orders, update status, cancel
  ✅  CouponController   — CRUD coupons, apply coupon, toggle active
  ✅  ReviewController   — add review, get reviews by product
  ✅  FeedbackController — submit feedback, list all feedback (admin)
  ✅  WishlistController — add/remove/list wishlist items
  ✅  CategoryController — list categories
  ✅  SseController      — real-time SSE events push (product/coupon updates)

  — SERVICES —
  ✅  UserService    — user registration and lookup
  ✅  ProductService — product search helpers
  ✅  EmailService   — order confirmation emails
  ✅  SseService     — broadcasting SSE events to connected clients
  ✅  CategoryService

  ⚠️  NO Spring Security — passwords stored in plain text in database
        Risk: Anyone with DB access can read all passwords
        Fix needed: BCrypt password hashing + Spring Security filter

  ⚠️  NO JWT / Session tokens — auth is only via localStorage role check
        Risk: Any user can change localStorage to "ADMIN" and access admin
        Fix needed: JWT token issued on login, validated on every API call

  ⚠️  NO input validation — no @Valid or @NotBlank on entity fields
        Risk: Empty products, null prices, blank emails can be saved
        Fix needed: Bean Validation annotations + global exception handler

  ⚠️  NO global exception handler (@ControllerAdvice)
        Risk: Raw Spring error stack traces exposed to frontend
        Fix needed: @RestControllerAdvice returning clean JSON error responses

  ❌  NO payment gateway integration (Razorpay / PayPal / Stripe)
        Currently: Orders are placed without any real payment
        Fix needed: Integrate Razorpay test API for payment before order

  ❌  NO order items table in database
        Currently: Cart items are stored as a String/JSON blob in Order
        Fix needed: Separate OrderItem entity (orderId, productId, qty, price)

  ❌  NO stock deduction on order placement
        Risk: Overselling — 10 users can order last 1 item
        Fix needed: Deduct stockQuantity in OrderController when order placed

  ❌  NO email credentials security
        Risk: Gmail password is exposed in application.properties plain text
        Fix needed: Use environment variables or application-secrets.properties

  ❌  NO pagination on product/order APIs
        Risk: 1000+ products will slow down entire page
        Fix needed: Pageable in Spring JPA + page/size params in API

  ❌  NO image storage system
        Currently: Images stored locally in /images folder
        Risk: Files lost on server restart, no CDN
        Fix needed: Cloudinary or AWS S3 for image hosting

  ❌  NO coupon expiry date
        Currently: Coupons only have isActive flag
        Fix needed: Add expiryDate field to Coupon entity
